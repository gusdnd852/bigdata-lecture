02. 머신러닝의 종류

머신러닝의 여러가지 종류에 대해 알아봅시다.

  • 1. 지도 학습 (Supervised Learning)
    • 1.1. 분류 (Classification)
    • 1.2. 회귀 (Regression)
  • 2. 비지도 학습 (Unsupervised Learning)
    • 2.1. 군집화 (Clustering)
    • 2.2. 차원축소 (Dimensional Reduction) + 시각화 (Visualization)
    • 2.3. 생성모델 (Generative Model)
  • 3. 강화학습 (Reinforcement Learning)
  • 4. 그 외의 학습 방식들 (Others)
    • 4.1. 준지도학습 (Semi-Supervised Learning)
    • 4.2. 자기지도학습 (Self-Supervised Learning)
    • 4.3. 전이학습, 사전학습 (Transfer Learning, Pretraining)
    • 4.4. 퓨샷학습 (Few-Shot Learning)
    • 4.5. 제로샷학습 (Zero-shot Learning)

  • 머신러닝엔 크게 3가지 분류가 있고 자세하게 구분하면 그보다 많게 분류할 수 있습니다.
  • 3가지 대분류인 지도학습, 비지도학습, 강화학습에 대해 알아봅시다.
  • 더욱 세밀한 분류인 전이학습, 준지도학습, 자기지도학습, 퓨샷학습, 제로샷학습에 대해서도 알아봅시다.

1. 지도 학습 (Supervised Learning)

지도학습은 데이터와 정답이 제공되는 경우를 말합니다. 그래서 "지도 (Superviesed)"라고 합니다. 위 그림처럼 사람이 직접 "이 동물은 기린이야. 이 동물은 고양이야" 처럼 사람이 컴퓨터를 지도하는 방법입니다. 다음과 같은 데이터셋의 형태라면 지도학습을 적용할 수 있습니다.

1.1. 분류 (Classification)

데이터를 보고 그 데이터가 속한 클래스를 정확하게 맞추는 문제를 분류문제 (Classification)이라고 하며, 분류문제는 지도학습의 가장 대표적인 예시입니다. 응용분야는 문서분류(Document Classification), 이미지분류(Image Classification), 암진단(분류, Cancer Diagnosis), 감정분류(Sentimental Analysis) 등이 있습니다. 아래 예시를 통해 자세히 배워봅시다.

참고로 기본적인 내용이지만 표를 볼 때, [이름, 신장, 몸무게, 비만] 등의 세로 줄을 '열' 혹은 'column'이라고 합니다. 또한 0번사람, 1번사람, ... 처럼 가로줄을 '행' 또는 'row'라고 합니다. 만약 이렇게 기본적인 내용이지만 모르는 내용이 있다면 언제든지 질문해주세요.

#collapse-hide
from matplotlib import pyplot as plt


def draw_weight_graph(data, boundary, title):
    """
    정상과 비만 데이터를 그래프로 표시하는 함수입니다.

    :param data: 표시할 데이터프레임
    :param boundary: 데이터 클래스를 구분하는 선
    """

    normal_people = data[data['비만(정답)']=='정상']
    fat_people = data[data['비만(정답)'] == '비만']
    qeury_person = data[data['비만(정답)'] == '???']

    plt.scatter(normal_people['신장'], normal_people['몸무게'], label='normal') 
    # 정상 데이터
    plt.scatter(fat_people['신장'], fat_people['몸무게'], label='fat') 
    # 비만 데이터
    plt.scatter(qeury_person['신장'], qeury_person['몸무게'], label='question') 
    # 쿼리 데이터 (맞출 문제)
    plt.plot(*boundary) 
    # 데이터 클래스를 구분할 수 있는 선

    plt.title(title)
    plt.xlabel('height')
    plt.ylabel('weight')
    plt.legend(loc='lower right')
    plt.grid(True)    
    plt.show()
    plt.close()

1.2. 회귀 (Regression)

회귀라고 하니까 뭔가 굉장히 어려워 보이지만 실은 하나도 그렇지 않습니다. 분류는 정답 클래스가 확실하게 구분된 경우에 해당합니다. 회귀는 지도학습으로 정답은 주어지지만, 정답 클래스가 정확하게 구분되지 않는 경우를 의미합니다.

  • 집의 가격 회귀 예시

#collapse-hide
def draw_housing_graph(data, boundary, title):
    """
    집의 넓이-가격 데이터를 그래프로 표시하는 함수입니다.

    :param data: 표시할 데이터프레임
    :param boundary: 데이터 클래스를 구분하는 선
    """

    fig = plt.figure()
    
    plt.scatter(data['넓이(m^2)'], data['가격(억)'] / 10) 
    plt.plot(*boundary) 
    plt.title(title)
    plt.xlabel('total floor area')
    plt.ylabel('price (bilion)')
    plt.grid(True) 
    plt.show()
    plt.close()

그래프를 보면 집의 가격과 면적이 정비례하는 것을 알 수 있고 우리는 이를 통해 하나의 가설을 세워서 집의 크기가 주어지면 가격을 대략적으로 예측해볼 수 있습니다.

#collapse-hide
def get_weight_bias(a1, a2):
    x1, y1 = a1
    x2, y2 = a2
    
    w = (y2 - y1) / (x2 - x1) # 기울기
    b = y1 - (w * x1) # Y 절편
    return w, b


def draw_housing_graph_with_sample(data, title, sample):
    """
    집의 넓이-가격 데이터를 그래프로 표시하는 함수입니다.

    :param data: 표시할 데이터프레임
    :param boundary: 데이터 클래스를 구분하는 선
    """

    plt.scatter(data['넓이(m^2)'], data['가격(억)'] / 10)
    a1 = (data['넓이(m^2)'].min(), data['가격(억)'].min() / 10) # 최소점
    a2 = (data['넓이(m^2)'].max(), data['가격(억)'].max() / 10) # 최대점
    plt.plot([a1[0], a2[0]], [a1[1], a2[1]])

    w, b = get_weight_bias(a1, a2)
    w, b = round(w, 3), round(b, 3) # 반올림
    
    predict = w * sample['넓이(m^2)'] + b
    plt.scatter(sample['넓이(m^2)'], predict)
        
    plt.title(title)
    plt.xlabel('total floor area')
    plt.ylabel('price (bilion)')
    plt.grid(True)
    plt.show()
    plt.close()
    
    print('예상 가격은 {} (억)입니다.'.format(predict * 10))
    print('찾아낸 직선의 방정식 : y = {w}x + {b}'.format(w=w, b=b))

  • 회귀 : 데이터와 정답을 가지고 학습쉬킨 뒤, 데이터만 줬을 때 [-∞, +∞]의 연속적인 값에서 임의의 값을 예측하내는 것을 의미합니다.

2. 비지도 학습 (Unsupervised Learning)

비지도학습은 지도학습과 달리 데이터와 정답이 제공되지 않는는 경우를 말합니다. 그래서 "비지도 (Unsuperviesed)"라고 합니다. 비지도학습에서는 정답열(라벨) 없이 데이터의 특성을 찾아서 원하는 작업을 수행하는데, 대표적으로 군집화, 차원축소(시각화), 생성모델 등이 있습니다. 지도학습은 주로 분류와 회귀 정도가 있지만, 비지도학습의 종류는 매우 다양합니다. 사실, 정답열(라벨)이 없는 데이터를 이용한 모든 작업을 비지도학습이라고 할 수 있습니다.

2.1. 군집화 (Clustering)

군집화는 데이터의 정답열(라벨)을 모를 때, 데이터의 특징을 가지고 데이터를 K개의 군집으로 나누는 것을 의미합니다. 나뉘어진 데이터가 분류의 클래스를 나타내는 것은 아니지만, 라벨 없이도 데이터를 군집으로 나눌 수 있습니다.

  • 붗꽃 데이터 군집화(Clustering) 예시

#collapse-hide
from sklearn.datasets import load_iris

iris_dataset = load_iris()
data, target = iris_dataset['data'], iris_dataset['target']

plt.scatter(data[:,0], data[:,2])
plt.xlabel('sepal length')
plt.ylabel('petal length')
plt.title('IRIS Dataset')
plt.grid(True)
plt.show()
plt.close()

위는 꽃받침 길이(X)와 꽃잎 길이(Y)로 나타낸 붗꽃 데이터의 분포입니다. 그림을 보면 알 수 있지만, 아래쪽의 데이터와 위쪽의 데이터가 확연히 나뉘는 것을 알 수 있습니다. 이렇게 데이터의 형상을 보면 데이터의 정답(라벨)이 없더라도, 대략적으로 군집화 시킬 수 있습니다. 즉, 비슷한 위치에 있는 데이터들끼리 모아서 총 K개의 군집을 지을 수 있습니다. 아래 그림을 봅시다.

#collapse-hide
from sklearn.cluster import KMeans
import numpy as np


dataset = np.column_stack([data[:, 0], data[:, 2]])


def cluster(K):
    model = KMeans(n_clusters=K)
    model.fit(dataset)
    
    for i in range(K):        
        plt.scatter(dataset[model.labels_ == i, 0], dataset[model.labels_ == i, 1])

    plt.xlabel('sepal length')
    plt.ylabel('petal length')
    plt.title('IRIS Dataset (Clustering, K={})'.format(K))
    plt.grid(True)
    plt.show()
    plt.close()


#collapse-hide
# 2개의 군집으로 군집화
cluster(K=2)

이렇게 우리가 원하는 갯수로 데이터를 군집화 시킬 수 있습니다. 군집화 시킬 때, 우리는 전혀 라벨 데이터를 사용하지 않았습니다. 그냥 데이터를 보고 비슷한 위치의 데이터를 비슷한 데이터로 규정하고 군집으로 나눈 것 뿐입니다. 중요한 것은 우리가 이 데이터를 군집화 시킬 때 실제 라벨을 사용한 것이 아니기 때문에, 군집화된 결과가 실제 정답의 분류를 의미하는 것은 아닙니다.

실제로 이 붓꽃 데이터는 아래와 같은 3개의 라벨(붓꽃종류)이 존재합니다. 실제 라벨로 분류한 결과와 군집화 결과의 차이를 봅시다.

군집화는 뭔가 마법의 기법같지만, 그저 가까이 있는 데이터들끼리 모아놓은 것일 뿐이고, 실제 분류결과와는 분명히 차이가 있습니다. 그러나 이 세상 데이터가 모두 정답(라벨)열이 포함된 것은 아니기 때문에 실제로 데이터 분석시에 군집화 기법은 매우 자주 사용되는 테크닉입니다. 또한, 우리에게 데이터 라벨이 없는 경우, 군집화 기법을 사용해 가상의 라벨을 만들고 그 라벨로 지도학습을 시도해볼 수도 있습니다.

2.2. 차원축소 (Dimensional Reduction) + 시각화 (Visualization)

차원축소는 높은 차원의 데이터를 낮은 차원의 데이터로 변환합니다. 차원축소의 주된 이점은 아래와 같습니다.

  • 3차원 이상의 데이터가 존재할 때 그 데이터를 시각화할 수 없습니다. 이러한 이유로 데이터의 차원을 축소 할 수도 있습니다.
  • 데이터의 차원이 매우 높다면, 그 만큼 연산의 복잡성이 높아집니다. 이러한 이유로 데이터의 차원을 축소 할 수도 있습니다.
  • 이러한 데이터 차원축소 기술은 데이터를 압축하는 용도로도 사용합니다. 데이터를 압축하여 저장공간을 아낄 수도 있습니다.

차원이란 데이터가 가질 수 있는 방향을 이야기합니다. 1차원 데이터는 수직선 위에 표시 될 수 있으며, 2차원 데이터는 평면, 3차원 데이터는 공간에 표시 될 수 있습니다. 우리가 흔히 보는 데이터 형식에서는 컬럼(열)의 갯수가 곧 차원의 갯수와 동일합니다. 차원축소 역시 라벨은 필요하지 않고 오로지 데이터만 가지고 수행하기 때문에 대표적인 비지도 학습 방법이라고 할 수 있습니다. 데이터의 차원을 감소시키면 아래와 같은 효과들이 나타납니다.

  • PCA를 이용한 차원 축소 (2D → 3D)

PCA라는 기법을 이용하여 데이터를 축소하여 연산량을 줄일 수 있습니다.



  • PCA를 이용한 이미지 압축

또한 이미지 등의 파일을 압축하여 필요한 저장공간을 줄일 수 있습니다.



  • 높은 차원의 데이터를 시각화

일반적으로 자연어 데이터는 매우 높은 차원을 가집니다. 이러한 자연어 데이터를 2 ~ 3차원으로 감소시켜 시각화해서 눈으로 단어들의 관계를 확인 할 수 있습니다.



2.3. 생성모델 (Generative Model)

생성모델은 요즘 가장 핫한 비지도학습 방식이라고 할 수 있습니다. 정답(라벨)이 없는 데이터로 모델을 학습시킨뒤, 사용자의 입력을 받으면 학습했던 데이터와 비슷한 데이터를 직접 만들어냅니다. 생성모델은 이미지 쪽에서 가장 자주 활용되고 있으며, 음성합성(TTS)나 자연어 생성등의 기술에서도 활용되고 있습니다. 역시나 대부분의 생성모델은 라벨(정답)을 사용하지 않기 때문에 생성모델 역시 비지도학습으로 분류됩니다. 생성모델의 예시는 아래와 같습니다.

  • GAN을 이용한 이미지 스타일 변경, 가짜 얼굴 생성등의 이미지 생성

실존하지 않는 가짜 사람 얼굴 이미지를 생성해내거나 이미지의 스타일을 원하는대로 변경하는 등의 작업이 가능합니다.



  • GPT를 이용한 가짜 기사등의 자연어 생성

생성모델을 이용하여 기사나 소설 같은 글을 쓰는 인공지능도 만들 수 있습니다.



3. 강화학습 (Reinforcement Learning)

강화학습은 일반적으로 정제된 데이터가 아닌 특수한 환경에서 에이전트의 활동을 학습시킵니다. 에이전트란 특정한 사물체를 의미하는데 게임에서의 게임 캐릭터등을 말합니다. 이러한 게임 캐릭터가 특정한 목적을 달성할 때마다 보상을 지급하고, 이 보상이 최대화 되도록 캐릭터의 행동을 지속적으로 조정해나가는 방식을 이야기합니다. 강화학습의 경우 주로 게임등에 이용되는데, 이번 강의에서는 강화학습에 관련된 내용은 다루지 않을 예정입니다. (강화학습은 빅데이터 분석과는 다소 거리가 있고, 오히려 인공지능 영역에 가깝습니다) 강화학습의 대표적인 예시는 아래와 같습니다.



  • 게임 강화학습 (알파고)

구글 딥마인드의 알파고는 대표적인 강화학습 예시입니다. 이 때, 바둑에서의 점수를 보상으로 하여 어떻게 돌을 놓을지 학습합니다.



  • 게임 강화학습 (슈퍼마리오, 쿠키런)

바둑 말고도 다양한 게임을 강화학습으로 학습시킬 수 있습니다. 아래 두 게임에서는 이동한 거리와 코인 등을 보상으로 하여 어떻게 움직일지 학습합니다.


강화학습은 이러한 게임 이외에도 자율주행, 로봇공학등에서 다양하게 실험되고 있으며, 앞으로 발전 가능성이 무궁무진 한 머신러닝 학습방법입니다. 그러나 본 강의에서는 단 8일의 시간 밖에 주어지지 않기 때문에 빅데이터 분석과 거리가 먼 강화학습이나 유전 알고리즘 등은 제외하였습니다.



4. 그 외의 학습 방식들 (Others)

머신러닝 학습 방법은 크게 지도학습, 비지도학습, 강화학습으로 구분하지만, 더 세부적으로 구별하면 아래와 같이 다양하게 구분 할 수 있습니다. 이에 대해 간단하게 알아봅시다.

4.1. 준지도학습 (Semi-Supervised Learning)

준지도학습 기법은 위의 그림처럼 일부의 데이터만 정답이 존재하고, 나머지 데이터는 정답이 존재하지 않을때 사용하는 학습 방식을 통틀어 말하는 개념입니다. 매우 다양한 준지도학습 방법이 존재하는데, 그중 가장 유명한 것은 위에서 언급한 군집화 기법을 이용하는 것입니다. 정답(라벨)이 없는 데이터는 군집화를 이용해 정답(라벨)을 임의로 만들어낸뒤, 기존에 있던 데이터와 섞어서 사용할 수 있습니다. 또 요즘 굉장히 핫한 딥러닝에서 사용하는 자기지도학습(Self-Supervised Learning) 기법도 역시 유명한 준지도 학습 기법 중 하나입니다.

4.2. 자기지도학습 (Self-Supervised Learning)

딥러닝의 미래라고 불리는 자기지도학습 기법입니다. 자기지도 학습은 Unsupervised 데이터셋으로 모델이 스스로 정답셋을 만들어내고 스스로 학습하는 방법을 의미합니다. 가장 유명한 자연어처리 모델인 GPT와 BERT가 이런방식으로 학습합니다. Newyork Times나 Wikipedia같은 대량의 데이터셋은 사람이 직접 정답(라벨)을 달기 어렵습니다. GPT는 이런 데이터셋의 단어를 순서대로 맞춰보는 식으로 정답을 만들어서 학습하고, BERT는 스스로 임의의 단어를 비워놓고 그 단어를 정답(라벨)로 간주한 뒤, 빈칸에 들어갈 단어가 무엇일지 맞추는 식으로 학습합니다. 이렇게 하면 사람이 직접 정답(라벨)을 만들지 않아도 되기 때문에 대량의 데이터 학습에 매우 유용합니다. 이렇게 자기지도 학습은 정답이 달리지 않은 데이터로 먼저 모델을 학습 한 뒤에 후술할 전이학습 기법을 이용하여 정답이 있는 소량의 데이터셋을 이용해 학습하기 때문에 준지도 학습이라고 불립니다.

4.3. 전이학습, 사전학습 (Transfer Learning, Pretraining)

역시나 딥러닝의 미래라고 불리는 전이학습(사전학습) 기법입니다. 전이학습 기법은 이미 많은 데이터로 학습이 끝난 모델의 학습 결과를 불러와서 사용하는 방식을 말합니다. 이미 많은 데이터로 학습이 끝난 모델의 학습 결과를 새로 학습할 모델에 불러오고, 사용할 태스크에 맞춰서 소량의 데이터만 다시 학습합니다. 이렇게 하면 매우 적은 데이터만 가지고도 수준급의 성능을 가진 모델을 만들 수 있습니다. 이미지 쪽에서는 주로 ImageNet(1000만장의 이미지 데이터셋)으로 학습된 모델의 학습 결과를 인터넷에서 다운로드 받아서 불러온 뒤, 본인이 학습하고 싶은 소량의 데이터만 새로 학습하면 적은 데이터로도 아주 높은 성능을 얻을 수 있으며, 자연어 쪽에서는 주로 Wikipedia, Newyork Times등으로 Self Supervised Learning을 마친 모델의 학습 결과를 인터넷에서 다운로드 받아서 불러온 뒤, 본인이 학습하고 싶은 소량의 Superviesed 데이터만 새로 학습하면 역시 적은 데이터로도 아주 높은 성능을 얻을 수 있습니다.

4.4. 퓨샷학습 (Few-Shot Learning)

보통 머신러닝/딥러닝에는 데이터를 많이 모아야 잘 된다는 말이 있습니다. 그러나 현실세계에서 데이터를 모으는 것은 매우 어려운 일입니다. 퓨샷학습 기법은 단 몇장의 데이터로만 학습 해도 높은 성과를 내게 만드는 학습기법입니다. 보통 이미지 퓨샷학습의 경우 클래스당 5장씩, 10장의 데이터(서포트셋)만 모으고 학습하는 경우도 있습니다. 이렇게 적은 데이터만으로도 높은 성능을 보이게끔 학습하는 방식을 퓨샷학습 기법이라고 합니다.

4.5. 제로샷학습 (Zero-shot Learning)

제로샷 학습 기법은 본적 없는 데이터를 구별해내는 학습기법입니다. 위 사진처럼 원숭이, 사자, 호랑이 데이터만 가지고 학습했으면 그 이외의 데이터가 입력되면 기존에 입력된 데이터가 아니라 새로운 클래스의 데이터라고 말할 수 있어야합니다. 때문에 원숭이, 사자, 호랑이 데이터만 가지고 학습 한뒤, 이 세가지 종류가 아닌 고양이나 강아지 이미지가 입력되면 이 데이터는 기존에 학습하지 않은 새로운 클래스의 데이터라고 구별해내는 기법을 말합니다.


이제 다양한 학습 방법과 머신러닝 기법들의 예시를 배우게 되었습니다. 다음 강의에서는 선형대수학과 Numpy를 이용한 산술 프로그래밍을 학습하겠습니다.
