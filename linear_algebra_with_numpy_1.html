1. 선형대수란 무엇일까요?

선형대수는 뭔가 거창한 이름같아 보입니다. 선형대수학이란 무엇일까요? 우리는 선형대수학에서 "선형"과 "대수학"라는 단어의 의미에 대해 생각해볼 필요가 있습니다.

1.1. 선형(Linear)이란?

위키피디아의 설명에 따르면 선형성 또는 선형은 직선처럼 똑바른 도형, 또는 그와 비슷한 성질을 갖는 대상이라는 뜻으로, 이러한 성질을 갖고 있는 변환 등에 대하여 쓰는 용어입니다. 일반적으로 우리가 직선에 대해 생각할 때 $y = x + 3$이나 $y = 2x$와 같은 직선을 생각해볼 수 있습니다.

#collapse-hide
from matplotlib import pyplot as plt


plt.plot([x for x in range(0, 10)], [x + 3 for x in range(0, 10)], label='y = x + 3')
plt.plot([x for x in range(0, 10)], [x * 2 for x in range(0, 10)], label='y = 2x')
plt.legend(loc='lower right')
plt.show()
plt.close()

$y = x + 3$의 경우 입력(x)을 넣으면 거기에 3을 더한 값이 결과(y)가 됩니다. $y = 2x$의 경우 입력(x)을 넣으면 거기에 2를 곱한 값이 결과 (y)가 됩니다. 이렇게 선형적인 직선의 경우 곱하거나 더한, 즉 변화를 가한 딱 그만큼만 변화가 이루어집니다. 그러나 비선형적인 $y = x^2 + 3$에 생각해봅시다.

#collapse-hide
plt.plot([x for x in range(-10, 11)], [x ** 2 +3 for x in range(-10, 11)], label='y = x^2 + 3')
plt.legend(loc='lower right')
plt.show()
plt.close()

$y = x^2 + 3$의 경우 입력에 대해 우리가 변화를 준 값은 +3 밖에 없지만 $x=10$일 때 y값은 $10+3$인 $13$이 아니라 $10^2 + 3$인 $103$이 됩니다. 이렇게 비선형적인 곡선의 경우, 우리가 변화를 가한 것과 다르게 값이 변화하였습니다. 즉, 선형성이라는 것은 우리가 변화를 가한 만큼만 변화하는, 예측 가능한 시스템을 의미합니다. 대부분의 경우 위 처럼 직선적인 시스템을 선형적인 시스템이라고 합니다.

1.2. 대수(Algebra)란?

대수학, 대수의 '대'는 큰 것을 의미하는 것이 아니라, '대신하다'를 의미합니다. 그러니까, 위와 같은 선형시스템(직선)이 수를 대신하는 것이 바로 선형대수학입니다. 그러면 '수'는 뭘까요? 기본적으로 수의 본질은 숫자가 아닌 연산에 있습니다. 950123-1234567과 같은 주민등록번호나 010-1234-5678과 같은 전화번호는 숫자는 맞지만 이 숫자로 연산을 해버리면 그 결과는 아무런 의미가 없습니다. 때문에 주민등록번호나 전화번호는 숫자는 맞지만 수는 아닙니다.


피타고라스는 기원전 약 500년 전에 All is number라는 말을 남겼습니다. 연산만 가능하다면 모든 것이 수라는 것입니다. 꼭 숫자가 아니더라도, 수학에서는 점과 점을 더하거나 도형과 도형을 더하는 등의 연산도 존재합니다. 우리가 앞으로 배울 내용에 따르면 직선과 직선을 더할 수도 있고, 직선과 직선을 곱할 수도 있고, 직선과 직선을 뺄 수도 있습니다. 때문에 이 직선으로 수를 대신할 것입니다. 피타고라스의 말 처럼 연산이 가능하다면 숫자가 아닌 직선도 수가 될 수 있다는 것이고 그렇게 직선이 숫자를 대신하는 수학이 바로 선형대수학입니다. 정리하자면, 선형대수학은 직선(선형)이 숫자를 대신(대)하는 수학입니다. 우리는 앞으로 선형대수학을 공부해서 이 직선들을 이용해 연산을 수행할 것입니다.

2. 선형대수학을 배우는 이유

이제 우리는 이 직선을 가지고 연산을 수행할 것이라고 이야기했습니다. 그런데 이 선형대수학이 도대체 빅데이터 분석과 무슨 상관일까요? 아래의 데이터를 볼까요?

3. 벡터와 스칼라

선형대수학에서의 기본 조각은 바로 직선이라고 했습니다. 이 직선을 우리는 이제부터 벡터라고 부릅니다. 흔히들 벡터는 $\vec{a}$나 $\vec{x}$처럼 알파벳 위에 화살표를 표시합니다. 그리고 흔히 우리는 물리학시간에 벡터는 "크기와 방향이 있는 물리량"이라고 배웁니다. 그런데 이렇게 생각하면 너무 어렵게만 느껴지고, 공부하기 꺼려집니다. 한번 데이터 관점에서 접근해봅시다. 우리는 위의 0번 과일에 대해 이야기할 때 다음과 같이 표현할 수 있습니다.

# 딕셔너리로 표현
과일0 = {"당도":35.2, "수분함유량": 16.3}

# 리스트로 표현
과일0 = [35.2, 16.3]


이렇게 표현하고 나면, 과일0이 곧 [35.2, 16.3]과 같은 숫자의 리스트로 표현된다는 것을 알 수 있습니다. 이 숫자를 점으로 표현하면 다음과 같습니다.

#collapse-hide

보통 물리학에서는 화살표의 꼬리가 여기저기 자유롭게 움직일 수 있고, 크기와 방향만 같으면 동일한 벡터라고 말하는데요. 선형대수에서는 화살표의 꼬리가 원점을 벗어나는 일은 거의 없으니 그냥 이렇게 생각하셔도 무방합니다. 선형대수의 큰 장점은 동일한 벡터를 [35.2, 16.3]과 같이 수치적으로 표현할 수도 있고, 공간상에서 기하학적으로도 표현할 수 있다는 것입니다. 그리고 그 두가지 관점 사이의 관계를 이해하다 보면 선형대수의 중요한 특징이 매우 쉽게 드러나게 됩니다.

4. Numpy란?

이제 파이썬의 수학관련 패키지인 Numpy를 사용해서 이러한 벡터들을 표현하는 법에 배울 것입니다. 파이썬 자체의 리스트와 수학표현도 매우 뛰어난 편이지만, 벡터나 행렬이 커지고 수행해야할 연산이 복잡해지면 이를 순수 파이썬만으로 다루는 것은 매우 복잡하고 어렵습니다. 선형대수 개념을 배우면서 Numpy를 사용해서 각각 개념을 직접 구현해보도록 하겠습니다.

일반적으로 우리가 프로그래밍을 할때는 보통 Rule-base로 프로그래밍 해왔습니다. 때문에 우리는 보통 규칙과 조건들을 다루었습니다. 조건 A가 참이면 a를 실행하고, 조건 B가 참이면 b를 실행해라. 이런식으로 말이죠. 그러나 Numpy를 이용하는 산술 프로그래밍은 그 느낌이 조금 다르고 이질적이여서 처음에 많이 어렵게 느껴질 수 있습니다. 저도 머신러닝을 배우고 1년 정도는 이론만 공부했기 때문에 코드 한줄 못짰던 경험이 있습니다. 기존에 조건과 규칙을 주로 다루던 Rule-base프로그래밍과 달리 산술 프로그래밍은 벡터들과 행렬들을 자르고 붙이고 곱하고 뒤집고 이런식으로 진행이 됩니다. 조금 이질적인 느낌이 있어도 친절하게 설명해드릴테니 잘 따라와주시길 바랍니다. 

import numpy as np

기본적으로 numpy를 import할때는 위와 같이 합니다. 이렇게 import 문을 사용하여 numpy 라이브러리를 사용하겠다고 선언하고, numpy라는 이름이 길기 때문에 np라는 닉네임을 사용하도록 합니다. 그리고나서 간단한 Numpy 벡터를 하나 만들어봅시다.

vec_x = [35.2, 16.3]
# 파이썬 리스트 (벡터)

npvec_x = np.array(vec_x)
# 넘파이 어레이 (벡터)

npvec_x

array([35.2, 16.3])

위와 같이 np.array()함수에 파이썬 리스트를 넣으면 넘파이 벡터가 만들어집니다. numpy의 파워풀함은 아래 내용들을 설명하면서 추가로 알려드리겠습니다.

5. 벡터의 두가지 기본연산 : 벡터합과 스칼라곱

5.1. 벡터 합 : 두 벡터를 이어 붙이기

공간상에서 기하학적인 개념에서 본다면 두 벡터의 합은 한 벡터의 꼬리를 다른 벡터의 끝에 놓고 두 벡터를 잇는 새로운 벡터를 그리는 것과 동일합니다. 즉, $\vec{v}$를 통해 이동했다가 그 위치에서 $\vec{w}$의 방향으로 그 크기만큼 이동한다면 결과적으로 두 벡터의 합 벡터인 $\vec{v} + \vec{w}$의 끝 지점에 위치하게 됩니다.


이를 수치적인 관점에서 계산해본다면 각 차원(자리)값 들을 더해주면 결과 벡터가 됩니다. 즉 1번째 원소끼리 더한 값이 결과 벡터의 1번째 원소가 되고, 2번째 원소끼리 더한 값이 결과벡터의 2번재 원소값이 됩니다.


이 개념은 수직선과 굉장히 비슷한 개념입니다. 우리가 2 + 3을 수직선에서 생각할 때 0에서 오른쪽으로 2칸 이동하고 다시 3칸 이동하는 것과 동일하게 벡터는 단지 위아래 방향이 하나 더 추가된 것 뿐입니다. 이렇게 벡터의 합 연산도 본질적으로 정수, 유리수와 같은 숫자들과 본질적으로 동일합니다. 이제 이 벡터 합을 numpy를 이용해 구현해봅시다.

import numpy as np

npvec_v = np.array([1, 2])
npvec_w = np.array([3, -1])

npvec_addiation = npvec_v + npvec_w

npvec_addiation

array([4, 1])

numpy를 이용하면 +연산으로 매우 쉽게 두 벡터의 합을 구해낼 수 있습니다. 만약 파이썬 리스트를 서로 더했다면 이러한 결과가 아니라 아래와 같은 결과를 얻었을 것입니다.

pyvec_v = [1, 2]
pyvec_w = [3, -1]

pyvec_addition = pyvec_v + pyvec_w

pyvec_addition

[1, 2, 3, -1]

Python 리스트의 합은 두 리스트의 원소를 모두 가지고 있는 새로운 리스트를 만드는 연산입니다. 때문에 이러한 벡터 연산을 편하고 깔끔하게 수행하려면 numpy를 사용하는것이 굉장히 좋습니다.

5.2. 스칼라 곱 : 벡터를 상수배 만큼 늘리거나 줄이기

스칼라곱은 벡터의 방향을 바꾸지 않고 그 크기만 상수배 만큼 늘리거나 줄이는 것을 의미합니다. 이러한 과정을 Scaling이라고 부르고, Scaling을 얼마나 할 것인지 결정하는 상수 값을 Scaler라고 합니다. 영어 발음으로는 '스케일러'가 맞지만 보통은 '스칼라'라고 부릅니다.


물론 음수를 곱하게 되면 방향이 뒤바뀌게 되지만 절대적으로 x와 y사이의 비율, 즉 기울기는 그대로 유지되기 때문에 방향이 변하지 않는다고 한 것입니다. [1, 2]에 -3을 곱해서 [-3, -6]이 된다고 해도 x와 y 사이의 비율은 그대로 1 : 2가 됩니다. 스칼라곱도 마찬가지로 수직선에서 생각할 수 있습니다. 숫자 3 위치에 어떤 값이 있다면, 그 값을 2배 늘리면 6에 도달하게 됩니다. 벡터의 경우는 방향만 추가되었을뿐 그 본질은 같습니다.


스칼라곱을 수치적으로 계산할 때는 벡터의 모든 차원에 스케일러 값을 곱해주면 됩니다. 벡터가 [3, 1]라면 여기에 2를 곱하면 [6, 2]가 됩니다. 간단하죠? 이제 스칼라 곱을 numpy를 이용해 구현해봅시다.

import numpy as np

npvec_v = np.array([3, 1])
scaler = 2

npvec_multiplication = npvec_v * scaler

npvec_multiplication

array([6, 2])

위와 같이 단순하게 * 연산을 사용하여 상수값 2를 numpy 벡터에 곱해주면 손쉽게 스칼라 곱을 계산해낼 수 있습니다. 만약 numpy가 아니라 파이썬 리스트를 곱했다면 아래와 같이 수행될 것입니다.

pyvec_v = [3, 1]
scaler = 2

pyvec_multiplication = pyvec_v * scaler

pyvec_multiplication

[3, 1, 3, 1]

파이썬리스트의 곱셈 연산은 해당 리스트를 n배로 복제해낸 원소를 가지고 있는 리스트를 새로 만드는 연산입니다. 이렇게 선형대수 산술 프로그래밍을 할 때는 numpy를 사용하는 것이 파이썬 리스트에 비해 압도적으로 편리하다는 것을 알 수 있습니다.

앞으로 이야기 하겠지만 선형대수는 위의 두가지 연산에서만 도는 경향이 있습니다. 선형대수에서 다루는 시스템인 직선이 바로 선형성을 유지해야하기 때문입니다. (제곱하거나 루트를 씌우거나 미적분을 하거나 하면 선형성이 깨져버리기 때문) 딱 더한만큼만 더해지고, 딱 곱한만큼만 곱해진 시스템만 다루기 때문에 벡터 합과 스칼라 곱. 이 두가지 연산이 가장 중요합니다.
